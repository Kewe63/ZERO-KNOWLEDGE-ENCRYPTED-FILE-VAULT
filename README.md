Live Demo: [https://zero-knowledge-encrypted-file-vault.vercel.app/](https://zero-knowledge-encrypted-file-vault.vercel.app/)

---

## 🔒 What it does

Files are encrypted **client-side in the browser** using AES-256-GCM before upload — the network never sees raw data. Once encrypted, the payload is committed to the blockchain, and storage operations are managed and tracked using a connected Aptos standard wallet (e.g., Petra).

Users can share files securely via a unique **File ID** and a **Decryption Key** pair.

## ✨ Features

- **Web3 Wallet Integration:** Seamless connection with Aptos wallets (Petra, vb.) using the AIP-62 standard.
- **Client-Side Encryption:** Military-grade AES-256-GCM encryption natively using the Web Crypto API.
- **Decentralized Storage:** Interacts with the Shelby SDK to create blob payloads on-chain.
- **Auto-Generated Passwords:** PBKDF2 derived keys (200,000 iterations) with built-in password generation.
- **Modern "Hacker" Interface:** Sleek, matrix-style UI with dynamic feedback.
- **Chunked File Processing:** Safely handles large file conversions locally without crashing the browser execution stack.

## 🛠️ Tech Stack

- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3

  - `@shelby-protocol/sdk`: For decentralized data blob payload creation.

## 🚀 Getting Started (Local Development)

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed along with `npm` (or `yarn`). Also, you will need a compatible Aptos Wallet like **[Petra](https://petra.app/)** installed in your browser.

### 1. Clone & Install
Clone the repository and install the NPM dependencies:
```bash
git clone https://github.com/Kewe63/ZERO-KNOWLEDGE-ENCRYPTED-FILE-VAULT.git
cd ZERO-KNOWLEDGE-ENCRYPTED-FILE-VAULT
npm install
```

### 2. Run Local Dev Server
Start the Vite development server:
```bash
npm run dev
```
Open `http://localhost:5173` in your browser.

### 3. Build for Production
To create an optimized final build for deployment:
```bash
npm run build
# Files will be bundled into the '/dist' directory.
```

## 🛡️ Privacy & Security

- **Zero-Knowledge:** Encryption happens 100% on the user's local machine.
- **Verifiable:** Transactions are registered directly using the connected wallet, establishing clear ownership.
- **Unreadable by Hosts:** The server (Shelby Protocol nodes) only stores and serves the raw encrypted bytes. Without the decryption key sent out-of-band by the uploader, the data is useless.
